## NextDocs Documentation

When creating or modifying documentation, read and follow the conventions in `.github/nextdocs-conventions.md`.

Key rules:
- Use `lowercase-with-hyphens` for all file and directory names
- Every directory needs a `_meta.json` for navigation — never include "index" in it
- Every section needs an `index.md` as its landing page
- Use Lucide icon names (e.g., Rocket, BookOpen, Settings) in `_meta.json`
- Store images in `_img/` directories next to markdown files
